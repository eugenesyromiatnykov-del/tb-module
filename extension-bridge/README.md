# Модуль ТБ — Chrome bridge

Розширення для Chrome, що додає на сторінку пацієнта в medics.ua блок «📋 Модуль ТБ» зі статусом, останньою флюоро та кнопкою оновлення.

## Встановлення локально (без публікації)

1. Chrome → `chrome://extensions/`
2. Увімкнути **Developer mode** (верхній правий кут)
3. **Load unpacked** → вибрати папку `extension-bridge/`
4. Розширення зʼявиться в списку. Натисніть на іконку → **Опції** (або клацніть на самому розширенні в полі options page) → введіть URL модуля та PIN → **Зберегти і перевірити**

## Як це працює

- Content script `tb-module-bridge.js` інʼєктиться на сторінки `medics.ua/*`
- Витягує Medics ID із URL або DOM
- Викликає `GET /api/extension-sync?medics_id=…` з заголовком `Authorization: Bearer <PIN>`
- Якщо пацієнт є — показує його статус та останню флюоро
- Якщо немає — кнопка «Додати в модуль ТБ»
- Кнопка «Оновити з МІС» робить `POST /api/extension-sync` з даними зі сторінки

## Адаптація до DOM medics.ua

Базова реалізація — мінімальна, бо точна структура `medics.ua` залежить від ділянки. Перегляньте і доадаптуйте дві функції в `tb-module-bridge.js`:

### `extractMedicsId()`

Пробує по черзі:
1. URL pattern `/patients/<id>` (відрегулюйте regex під реальну схему)
2. Текстова мітка «Medics ID» з сусіднім номером
3. `[data-medics-id]` атрибут на елементі

### `extractPatientContext()`

Зчитує `data-tb-field="surname"`, `data-tb-field="first_name"` тощо. У medics.ua, ймовірно, таких атрибутів немає — додайте свої селектори.

## Auth

PIN зберігається в `chrome.storage.sync` (синхронізується між пристроями користувача, шифрується Google'ом). Не передається на сервери третіх сторін. У запитах до модуля передається у заголовку `Authorization: Bearer <PIN>`.

Сервер `/api/extension-sync.ts` робить `bcrypt.compare(pin, PIN_HASH)` — той самий хеш, що використовується для входу в веб-додаток.

## Безпека

- HTTPS обовʼязковий (extension не може робити запити з `http://medics.ua` через mixed content)
- CORS налаштовано на серверній стороні: `chrome-extension://*` та `medics.ua` пропускаються
- При помилці 401 розширення показує «Помилка авторизації» — введіть PIN заново у налаштуваннях
